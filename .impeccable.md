## Design Context

### Users
Mixed audience: recruiters/hiring managers evaluating candidates, fellow developers networking, potential clients considering collaboration. They browse quickly on desktop and mobile, need to immediately understand who Hafizuddin is and what he can build.

### Brand Personality
Clean, minimal, focused. The portfolio should project quiet confidence — let the work and credentials do the talking. Not flashy, not corporate, not dark-mode-with-neon. Sharp and deliberate.

### Aesthetic Direction
Light mode primary (system dark mode supported). White with tinted neutrals toward indigo/purple primary. Strong typographic hierarchy using Sora (display) + Figtree (body). Bold dark borders for crisp definition. One accent color used sparingly. No gradient text, no glassmorphism overuse, no identical section templates repeated across every section.

Theme: Predominantly light — the audience reviews this during working hours in a bright context.

### Design Principles
1. **Typography carries hierarchy** — size and weight contrasts, not gradients or glow effects
2. **Color is used sparingly** — primary on interactive elements, tinted neutrals everywhere else
3. **Sections feel distinct** — each section has its own visual treatment, not one cookie-cutter header pattern stamped on everything
4. **Interactions are subtle** — hover states signal affordance, not demonstrate capability
5. **Content over decoration** — decorative elements exist only when they serve communication

### Anti-References
- Generic AI-generated portfolio (gradient text, identical badge headers, glassmorphism cards)
- Corporate blandness (too safe, no personality)
- Heavy dark mode + neon (purple glow aesthetic)
